# Telegram Data-Bot (C# / .NET)

Интерактивный Telegram-бот для автоматизации сбора данных и управления пользователями. Проект демонстрирует навыки работы с внешними API, асинхронным программированием и интеграцией с реляционными базами данных (MS SQL Server).

## Ключевой функционал
*   **Регистрация пользователей**: Автоматический сбор и сохранение данных пользователя (Chat ID, Username) в базу данных при первом взаимодействии.
*   **Интеграция с СУБД**: Выполнение SQL-запросов «на лету» для получения актуальных списков позиций из MS SQL Server.
*   **Интерактивное меню**: Реализация пользовательского интерфейса через `ReplyKeyboardMarkup` для удобной навигации без ввода текстовых команд.
*   **Обработка обновлений**: Использование механизма Long Polling с отслеживанием `offset` для надежной доставки сообщений.

## Технологический стек
*   **Язык**: C# (.NET).
*   **Библиотека**: `Telegram.Bot` (основана на асинхронном подходе Task-based Asynchronous Pattern).
*   **База данных**: MS SQL Server (ADO.NET / SqlClient).
*   **Протоколы**: JSON API, HTTPS.

## Технические особенности реализации
*   **Asynchronous First**: Весь сетевой ввод-вывод и запросы к БД реализованы через `async/await`, что предотвращает блокировку основного потока.
*   **Безопасность соединений**: Использование защищенных строк подключения (`Trusted_Connection`, `TrustServerCertificate`) для работы в различных средах.
*   **Отказоустойчивость**: Реализован бесконечный цикл обработки событий с блоками `try-catch` для защиты бота от падения при сетевых сбоях или ошибках API.

## Инструкция по настройке

1.  **Конфигурация БД**: 
    Измените переменную `connectionString` в файле `Program.cs`, указав данные вашего сервера MS SQL.
2.  **Токен бота**:
    Замените константу `TOKEN` на значение, полученное от `@BotFather`.
3.  **Запуск**:
    ```bash
    dotnet restore
    dotnet build
    dotnet run
    ```
